RomânăSimplă Language Specification

Working draft for a simplified and regularized Romanian.

Author: Draft workspace scaffold
Version: 0.1.0-draft
Date: March 2026

Abstract

RomânăSimplă is a formal simplification proposal for Romanian. The project does not assume that Romanian should be forced into English-style reforms. Instead, it starts from Romanian as it is actually used and asks which grammatical, orthographic, and syntactic burdens can be reduced without destroying clarity or recognizability.

This draft is intentionally narrow. The goal of version 0.1.0 is to define a stable experimental core that can be tested on a small corpus before larger claims are made.

1. Design Principles

Principle Target Draft Direction
Orthographic clarity Reduce typing friction without harming readability Keep standard Romanian as the canonical layer; define an optional ASCII fallback
Morphological regularity Reduce conjugation and agreement burden Prefer explicit markers over many inflectional variants where practical
Parser clarity Limit ambiguity introduced by simplification Avoid adding symbolic rules unless they improve determinism
Native continuity Preserve Romanian identity Reuse familiar Romanian roots, word order, and core function words
Measurability Keep the system testable Validate on a fixed Romanian corpus before expanding rules

2. Open Design Decisions

This section defines the core questions that must be settled before the system can move from concept to specification.

2.1 Orthography

Questions:

  • Should ă, â, î, ș, and ț remain mandatory in the canonical form?
  • What is the ASCII fallback, if any?
  • Should the simplified system preserve current Romanian spelling entirely, or regularize a small number of high-friction spellings only?

Draft position:

  • Canonical layer keeps standard Romanian orthography.
  • Secondary ASCII mode may be defined for typing convenience.
  • Any spelling reform should be minimal and evidence-based.

2.2 Articles and Definiteness

Questions:

  • Should the enclitic definite article remain attached to the noun?
  • Should definiteness become a separate particle?
  • How should indefinite marking be handled?

Draft position:

  • Treat article handling as a primary simplification candidate.
  • Compare current Romanian forms with a detached-article model on corpus data.

2.3 Verbs

Questions:

  • Which tense, mood, and person distinctions are essential?
  • Can person marking be reduced if pronouns are explicit?
  • Can auxiliaries replace a large portion of inflectional complexity?

Draft position:

  • Verb simplification is likely the highest-value reform area.
  • Start with present, past, future, imperative, and negation behavior.

2.4 Pronouns and Clitics

Questions:

  • Are clitic objects preserved, reduced, or replaced by full analytic forms?
  • How should politeness distinctions such as tu and dumneavoastră be handled?
  • Is null-subject behavior preserved?

Draft position:

  • Prefer explicitness over clitic complexity in early drafts.
  • Keep politeness distinctions unless they create unacceptable structural cost.

2.5 Gender, Number, and Agreement

Questions:

  • Which agreement rules are essential for comprehension?
  • Should gender be preserved lexically but reduced grammatically?
  • Can plural and adjective agreement be simplified without severe ambiguity?

Draft position:

  • Preserve distinctions that carry clear semantic value.
  • Challenge distinctions that mainly create memorization burden.

3. Candidate Reform Tracks

These are candidate directions, not adopted rules.

Track A: Conservative Simplification

  • Keep standard spelling.
  • Reduce verb paradigms.
  • Regularize clitic behavior.
  • Preserve most Romanian surface appearance.

Track B: Analytic Romanian

  • Move more grammar into explicit particles.
  • Reduce inflection more aggressively.
  • Make subject, tense, and definiteness more overt.

Track C: Typing-Optimized Romanian

  • Keep canonical written form.
  • Add strict ASCII typing mode.
  • Optimize for keyboards and digital communication rather than speech reform.

4. Minimal Version 0.1 Rule Set

The first version should stay small enough to test honestly.

Recommended rule count for 0.1.0:

  1. One orthography policy.
  2. One article policy.
  3. One negation policy.
  4. One present-tense simplification rule.
  5. One past/future strategy.
  6. One pronoun/clitic policy.
  7. One agreement-reduction policy.

5. Corpus Method

The corpus should include:

  • conversational sentences
  • informational prose
  • instructions
  • questions
  • negation
  • possession
  • temporal reference
  • definiteness contrasts
  • pronoun and clitic use

Evaluation dimensions:

  • byte length
  • token count
  • ambiguity introduced
  • ambiguity removed
  • reading familiarity
  • typing effort

6. Risks

  • Romanian may yield less compression than English because its orthography is already relatively efficient.
  • Over-simplifying clitics or agreement may make the language feel less Romanian, not more usable.
  • A system can become regular yet harder to process if too many new markers are introduced.

7. Next Steps

  1. Build a 50-100 sentence Romanian baseline corpus.
  2. Choose one candidate reform track.
  3. Draft explicit rules for articles, verbs, and clitics.
  4. Convert 10 sample sentences.
  5. Review the result critically before expanding scope.
